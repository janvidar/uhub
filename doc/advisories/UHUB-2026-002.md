# UHUB-2026-002 — Multiple remotely-triggerable memory-safety crashes in the ADC parser and login path

- **Severity:** High
- **Attack vector:** Network. Some pre-authentication; the remainder reachable by
  any peer that can send an `INF`/`BINF` (i.e. trivially reachable on any open hub).
- **Impact:** Denial of service (hub crash); out-of-bounds read/write, integer
  over/underflow, NULL-pointer dereference
- **Affected:** uhub <= 0.5.0 (and all pre-release builds before 0.6.0)
- **Fixed in:** 0.6.0

## Description

A set of bounds- and integer-handling defects in the ADC wire-format parser
(`src/adc/message.c`, `src/adc/sid.c`) and the login pipeline (`src/core/inf.c`)
allow a remote peer to crash the hub. Because uhub is single-threaded, any one
crash is a full-service DoS.

> **Note:** the user-agent heap-write defect (`7782f7d`) was originally tracked
> here as item 002-a. On closer analysis it is an attacker-controlled heap write
> (code-execution candidate), not a read/crash, and has been split out into its
> own advisory — see **[UHUB-2026-004](UHUB-2026-004.md)**.

| ID | Component | Defect | Fix |
|----|-----------|--------|-----|
| 002-b | `inf.c` `set_feature_cast_supports()` | Out-of-bounds **read** while parsing INF feature tokens | `3eb13a3` |
| 002-c | `message.c` `adc_msg_parse()` | Negative array index (`cache[-1]`) in the argument-offset check | `12b653a` |
| 002-d | `message.c` `adc_msg_terminate()` | Out-of-bounds read `cache[(size_t)-1]` on a zero-length message | `4ac65cd` |
| 002-e | `message.c` `adc_msg_remove_named_argument()` | Integer **underflow** of message length (size wraps) | `061271a` |
| 002-f | `message.c` `adc_msg_escape_length()` | Integer **overflow** (`int` length math) | `438b102` |
| 002-g | `message.c` named-argument helpers | Dereference when `arg_offset == -1` | `d1c1315` |
| 002-h | `ipcalc`/number parsers | Signed-overflow undefined behaviour in number/IP parsing | `c533661` |
| 002-i | `sid.c` `sid_alloc()` | Crash in the SID pool allocator when exceeding 1M SIDs (out-of-range map index) | `4a7bc70` |
| 002-j | `sid.c` `sid_free()` | Out-of-bounds / double-free on invalid SID | `fe45404` |
| 002-k | `message.c` | NULL-pointer dereference on malformed input | `5e14745` |

Items 002-c, 002-d, 002-f, 002-g, 002-h sit on the same pre-authentication parse
path as UHUB-2026-001. Item 002-b is reached during INF processing, i.e. by any
client that completes far enough to send its info — effectively unauthenticated
on an open hub.

## Remediation

Upgrade to **uhub 0.6.0** or later. All listed defects are fixed; the parser
changes are covered by regression tests in `autotest/test_message.tcc`.

## Workarounds

Restrict network exposure until upgraded. Running the hub built with stack/heap
hardening converts several of these from silent OOB into clean aborts but does
not prevent the DoS.
